# ZoomChartComponent

Этот компонент написан полностью ИИ потому что тут много математики над которой мне было лень думать.

Компонент для `ComposableHover`, управляющий областью отображения графика (`chart.setRenderBounds`):
пан мышью/тачем, зум колесом, пинч-зум, инерция и лимиты с опциональным «резиновым» оверскроллом (elastic).
Сам ничего не рисует. Функционал завершён — это финальное состояние, не закладывайте сюда архитектуру «на будущее».

## Использование

```ts
new ZoomChartComponent({
  chart,                       // UniversalChart
  zoom: true,                  // разрешить зум (колесо + пинч)
  panDirection: 'horizontal',  // 'horizontal' | 'vertical' | 'all' | false — какие оси активны
  deceleration: { mousePan: 0.99, touchPan: 0.99, touchZoom: 0.95 }, // опционально; значение вне (0,1) отключает инерцию
  limits: { minX, maxX, minDeltaX, maxDeltaX, /* ...Y */, elastic: true }, // опционально
})
```

Единственное место использования в проекте: `src/pages/infographics/pages/onslaught/leaderboard/components/detail/Charts.ts`.

## Модель

- **Обработчики событий только записывают состояние** (`pan` / `pinch` / `wheel`), никаких изменений bounds.
  Всё применяется одним пайплайном в `onBeforeLayout`: pinch → pan → wheel → шаг моушенов → проекция → `setRenderBounds`.
- **Raw vs displayed.** Жесты и инерция интегрируются в «raw» (несопротивляющемся) пространстве.
  Отображаемые границы — чистая проекция raw: тождественная внутри лимитов, rubber band снаружи (elastic)
  или clamp (hard). В hard-режиме raw наоборот жёстко клампится на каждом шаге применения.
- **Якорь проекции (anchorT)** — экранная доля, вокруг которой растягивается эластичный overzoom:
  середина пинча / курсор колеса во время зума, иначе 0.5. При смене якоря raw пересчитывается
  (`rebaseRawAnchor`), чтобы displayed не прыгнул.
- **Каналы.** Каждая ось анимируется двумя независимыми каналами: `center` (перенос) и `logRange` (зум).
  Поэтому, например, пружина возврата зума продолжает работать, пока по той же оси летит инерция пана.
- **Моушены.** Канал владеет либо инерцией (интегрирует raw напрямую), либо пружиной (следует кривой
  в displayed-пространстве, записывается обратно в raw через обратную проекцию). Пружина стартует только
  когда ввод, владеющий каналом, закончился.
- **Release** (`releaseAxis` → `decideChannelRelease`): если канал внутри лимитов или скорости хватит
  доскользить внутрь — инерция; иначе пружина к ближайшей валидной цели, со скоростью, переведённой
  в displayed через производную rubber band. `velocity: null` для канала = не трогать его текущий моушен;
  `inertia: null` = разрешены только пружины.
- **Инерция у границы**: пересекая лимит наружу, передаёт остаток скорости пружине, прижатой к границе
  (`inertiaBounceMotion`) — отскок. Во время активного жеста инерция не может оверскроллить (жёсткая остановка).

## Неочевидные места (не «чинить»)

- `wheel` — это батч: события копятся в `pending`, применяются в следующем `onBeforeLayout`;
  батч живёт `WHEEL_BATCH_TIMEOUT` после последнего события, чтобы после остановки колеса стартовала
  пружина возврата. Пинч всегда убивает wheel-батч.
- Сэмплеры скорости (`recordPanVelocity`, `samplePinchVelocity`) отбрасывают «bursts» — события,
  пришедшие сильно чаще среднего темпа: такие сэмплы накапливаются в следующий (baseline не сдвигается).
  Пауза дольше `INPUT_STOP_TIMEOUT` обнуляет скорость (палец остановился перед отпусканием).
- Пан сразу после пинча подавляет собственную инерцию (`suppressInertia`), пока не сдвинется на
  `PAN_SUPPRESS_RELEASE_DISTANCE` px — иначе дрожание пальцев при отрыве убивало бы инерцию пинча.
- Политики остановки моушенов различаются по вводу: `stopAllMotions` (пинч), `stopMotionsForPan`
  (тач-пан сохраняет зум-инерцию и пружину возврата зума), `stopMotionsForWheel` (сохраняет инерцию пана).
- В лимитах все сравнения с fp-допуском `LIMITS_EPSILON`: bounds постоянно раскладываются в
  (center, range) и собираются обратно, дрейф в несколько ULP не должен считаться нарушением.
- `clampAxisToLimits` возвращает входной объект без изменений, если ничего не нарушено — вызывающие
  сравнивают результат с входом по ссылке/значению, чтобы отличить реальный clamp.
